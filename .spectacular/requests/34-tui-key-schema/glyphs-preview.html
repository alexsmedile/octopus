Octopus glyphs — progress axis

Bucket position via arrow weight (·/▷/▶), progress via circle fill (○/◐/◑/●). Two variants below: combined (two cells, bucket + progress) and collapsed (one cell, the progress glyph alone — bucket inferred by color/context).

Variant 1 — Combined (2 cells: bucket + progress)

· draft the migration plan backlog · idle bucket=backlog
·○ draft the migration plan backlog · sketched progress=open
▷ queue the rollout next · idle bucket=next
▷○ queue the rollout next · started progress=open
▶○ fix the webhook auth bug now · just opened progress=open
▶◐ fix the webhook auth bug now · halfway progress=half
▶◑ fix the webhook auth bug now · most-done progress=most
● add focus-mode keymap done progress=done
✕ explore Go rewrite dropped progress=dropped
▶! deploy hotfix now · blocked run_state=blocked
▶? deploy hotfix now · waiting issue=waiting

Two cells per row. Bucket arrow is colored by bucket; progress glyph is colored by activity. Reads top-down like a kanban + Gantt at once. Cost: 2 char columns of glyph width instead of 1.

Variant 2 — Collapsed (1 cell: progress only, color = bucket)

· draft the migration plan backlog · idle color=backlog
○ draft the migration plan backlog · sketched color=backlog
○ queue the rollout next · open color=next
◐ queue the rollout next · half (started early) color=next
○ fix the webhook auth bug now · just opened color=now
◐ fix the webhook auth bug now · halfway color=now
◑ fix the webhook auth bug now · most-done color=now
● add focus-mode keymap done color=done
✕ explore Go rewrite dropped color=drop
! deploy hotfix now · blocked color=drop
? deploy hotfix now · waiting color=drop

One cell per row. Bucket disambiguated by color alone. Tightest layout, but loses bucket info for color-blind / monochrome readers.

Variant 3 — Hybrid (the recommendation)

Rows live inside a bucket pane already, so the bucket is implicit. Use the progress circle alone in row slot 1; reserve the arrow only for the "active session" state (`▶`) — the loudest signal.

· draft the migration plan idle / parked progress=none
○ queue the rollout open progress=open
◐ fix the webhook auth bug in progress (half) progress=half
◑ finalize the patch in progress (most) progress=most
▶ deploy hotfix session live session_id set
● add focus-mode keymap done progress=done
✕ explore Go rewrite dropped progress=dropped
! deploy hotfix blocked run_state=blocked
? deploy hotfix waiting issue=waiting

One cell, progress is the carrier, bucket comes from pane context (rows in the Backlog pane look different from rows in the Now pane). Loudest exception states (`▶` session, `!` blocked, `?` waiting, `✕` dropped) break out of the circle ladder so they pop visually.
